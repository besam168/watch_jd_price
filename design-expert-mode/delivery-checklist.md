# Delivery Checklist

每次设计交付前，默认检查以下项目：

## 通用
- [ ] 是否明确交付物类型
- [ ] 是否有一眼可见的主标题
- [ ] 是否有清晰层级
- [ ] 是否留白足够
- [ ] 是否对齐整齐
- [ ] 是否配色统一
- [ ] 是否风格一致
- [ ] 是否避免元素堆满

## PPT 专项
- [ ] 是否一页一重点
- [ ] 是否封面有主视觉与标题安全区
- [ ] 是否内页图文比例合理
- [ ] 是否章节节奏分明
- [ ] 是否便于后续修改

## 网页专项
- [ ] 是否首屏抓人
- [ ] 是否 CTA 清晰
- [ ] 是否内容顺序合理
- [ ] 是否考虑移动端
- [ ] 是否有信任证明区

## 原型 / UI 专项
- [ ] 是否信息架构清楚
- [ ] 是否交互流顺畅
- [ ] 是否按钮主次明确
- [ ] 是否组件复用一致
