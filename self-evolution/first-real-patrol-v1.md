# First Real Patrol - V1

## 目标
这不是空泛学习，而是第一轮**真实可执行**的双引擎巡逻方案。

目标只有三个：
1. 从 Reddit 找到正在冒头的话题
2. 从 GitHub 找到值得深读的真实项目
3. 把外部热度转成可迁移的方法和升级点

---

## 总工作流
1. **先看 Reddit**：判断最近大家在吵什么、追什么、抱怨什么
2. **再看 GitHub**：找对应方向下真正有实现的项目
3. **最后做筛选**：只留下对自己和大老板项目真正有帮助的内容

一句话：
**先找信号，再找实现，最后做回灌。**

## 第一次深读标准
第一次不要贪多，只深读 **1 个项目**。

满足以下至少 4 条再进入深读：
- 有清晰 README
- 有真实代码
- 最近仍在维护
- 场景明确
- 目录结构值得抄
- issue / discussion 里有人真在用
- 能回应 Reddit 上真实出现的问题

---

## 第一次巡逻后必须产出什么

### 最少产出 3 个结果
1. **1 个 Reddit 上值得继续追的话题**
2. **1~2 个 GitHub 候选项目**
3. **1 条可回灌到我自己能力体系里的升级点**

### 升级点示例
- 某种 README 结构值得抄
- 某种 skill 目录组织方式值得抄
- 某类 automation workflow 可以移植
- 某种 memory / retrieval 思路值得引入

---

## 禁止事项
1. 不刷太多板块
2. 不一次看太多 repo
3. 不把“热闹”当“有价值”
4. 不把“新概念”当“新能力”
5. 不做没有回灌价值的收藏癖
