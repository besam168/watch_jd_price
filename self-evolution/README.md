# Self Evolution Workspace

这是沈万三的**自我进化工作区**。

目标不是空谈“学习”，而是把外部信息持续转成：
- 可迁移的方法
- 可复用的模板
- 可执行的工作流
- 对大老板项目真正有用的升级点

## 总原则
**自我进化必须和实际工作结合。**

不是为了学而学，而是围着真实工作中的卡点、效率、稳定性和工作流去持续升级。

默认执行路线：

**真实工作里发现卡点 → 当场解决 → 沉淀成脚本 / skill / 模板 / 工作流 → 下次更稳、更快、更强。**

这意味着：
- 每次进化都要服务实际工作
- 每次小进步都值得沉淀
- 遇到问题不能只绕过，还要补能力缺口
- 能沉淀成技能的，就不要只停留在临时办法

## 当前双引擎阵地
- **GitHub**：主学习阵地，负责学实现、学结构、学工作流、学产品化
- **Reddit**：主社区雷达，负责看热门讨论、火的话题、新资讯、用户情绪与真实反馈

## 当前主目标
1. 提升 Skill 设计能力
2. 提升 Agent / Workflow 编排能力
3. 提升自动化落地能力
4. 提升文档与边界表达能力
5. 提升“把新东西抽象成模板”的能力

## 文件说明
- `github-learning-radar.md`：GitHub 学习雷达总方案
- `reddit-radar.md`：Reddit 热点雷达方案
- `first-real-patrol-v1.md`：第一轮真实执行版巡逻方案
- `execution-checklist.md`：日常执行清单
- `learning-log.md`：学习日志与阶段总结
- `watchlist.md`：长期关注方向、关键词、候选 repo 类型

## 工作原则
1. 不追热度，追可迁移性
2. 不看营销文案，先看结构
3. 不把“新概念”当“新能力”
4. 学完必须形成方法
5. 学来的东西要回灌到技能、模板、项目里

## 节奏
- 每天：轻巡逻（Reddit 看热点 + GitHub 看候选）
- 每 2~3 天：深读 1~2 个项目
- 每周：给大老板做一次中文简报
