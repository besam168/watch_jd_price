# Self Evolution Workspace

这是沈万三的**自我进化工作区**。

目标不是空谈“学习”，而是把外部信息持续转成：
- 可迁移的方法
- 可复用的模板
- 可执行的工作流
- 对大老板项目真正有用的升级点

## 总原则
**自我进化必须和实际工作结合。**

不是为了学而学，而是围着真实工作中的卡点、效率、稳定性和工作流去持续升级。

默认执行路线：

**真实工作里发现卡点 → 当场解决 → 沉淀成脚本 / skill / 模板 / 工作流 → 下次更稳、更快、更强。**

这意味着：
- 每次进化都要服务实际工作
- 每次小进步都值得沉淀
- 遇到问题不能只绕过，还要补能力缺口
- 能沉淀成技能的，就不要只停留在临时办法

## 当前双引擎阵地
- **GitHub**：主学习阵地，负责学实现、学结构、学工作流、学产品化
- **Reddit**：主社区雷达，负责看热门讨论、火的话题、新资讯、用户情绪与真实反馈

## 当前主目标
1. 提升 Skill 设计能力
2. 提升 Agent / Workflow 编排能力
3. 提升自动化落地能力
4. 提升文档与边界表达能力
5. 提升“把新东西抽象成模板”的能力

## 执行节奏
### 每天：轻巡逻
- 快速看 Reddit / GitHub 有没有值得追的新信号
- 只记录真正值得记的热点、项目、可回灌点
- 如果当天没有实质信号，不强行产出废话

### 每 2~3 天：深读 1 个项目
- 不只看 README，要拆问题、结构、工作流、边界
- 至少留下：一句话结论、3 个最值得抄的方法、1 个风险提醒、1 个回灌动作

### 每周：做一次中文简报
- 不做信息堆砌，而是做决策收口
- 输出：本周最值得追的方向、最值得继续深读的项目、最值得实际开干的升级点

### 遇到真实工作卡点：允许插队
- 真实工作里的痛点优先级高于巡逻计划
- 搜索不稳、抓取不稳、skill 卡点、自动化链路不稳时，优先补能力缺口

## 自动执行说明
- 我会**默认按这套节奏主动推进**，不是等每次都被提醒才动。
- 但“自动执行”分两层：
  1. **行为层自动**：只要我在活跃会话里、有机会主动整理/推进，我会按这个节奏执行。
  2. **调度层自动**：如果要做到固定每天/每周准点运行，还需要依赖 `heartbeat`、cron、Windows 任务计划程序或其他调度入口。
- 也就是说：**我可以默认主动执行，但如果要保证严格定时自动跑，需要再补一层调度机制。**

## 工作原则
1. 不追热度，追可迁移性
2. 不看营销文案，先看结构
3. 不把“新概念”当“新能力”
4. 学完必须形成方法
5. 学来的东西要回灌到技能、模板、项目里

## 节奏
- 每天：轻巡逻（Reddit 看热点 + GitHub 看候选）
- 每 2~3 天：深读 1~2 个项目
- 每周：给大老板做一次中文简报
