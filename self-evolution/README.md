# self-evolution

## 项目定位

`self-evolution` 不是一个给终端用户直接使用的成熟产品，也不是一个已经工程化完成的情报平台。

它当前更准确的定位是：

**一个面向 AI / 自动化 / Agent 方向的“双引擎外部巡逻原型”。**

这套原型的核心目标是：
1. 从 **Reddit** 发现正在冒头的话题、真实用户痛点和社区情绪
2. 从 **GitHub** 找到对应方向下值得深读的真实项目与实现
3. 把外部热点转成 **可回灌到自身项目的方法、结构和升级点**

一句话概括：

**Reddit 提题，GitHub 解题，最后回灌到自己的能力体系。**

---

## 这个项目有什么用

它的作用不是“自动帮用户完成某个业务”，而是帮助操作者建立一个稳定、可复制的外部学习与情报巡逻流程。

当前主要用途：

### 1. 持续发现值得追的方向
不是漫无目的刷网页，而是围绕固定主题持续观察：
- AI agent framework
- workflow / automation
- memory / RAG / retrieval
- browser automation
- desktop automation
- prompt / persona / template
- report / monitor / summary pipeline

### 2. 把社区热点和真实实现连起来
很多外部热点只停留在讨论层；这套方案的目标是：
- 先从 Reddit 找信号
- 再去 GitHub 找实现
- 最后筛出真正值得学习和迁移的方法

### 3. 为当前项目提供方法升级输入
它适合给以下类型工作提供外部输入：
- OpenClaw skill 设计
- agent orchestration
- 自动化采集与报告流水线
- memory / retrieval 方案
- browser / desktop automation
- prompt / template 系统设计

## 当前目录与文件说明

### 方法文档

#### `first-real-patrol-v1.md`
第一轮真实巡逻方案总说明。

作用：
- 定义第一轮巡逻目标
- 明确“先 Reddit、后 GitHub、再回灌”的总工作流
- 规定第一轮巡逻范围
- 给出最小输出模板和禁区

#### `execution-checklist.md`
执行节奏与交付清单。

作用：
- 定义“日常轻巡逻 / 深读 / 周报”三层节奏
- 规定每层该看什么、记什么、输出什么
- 避免陷入无效收藏和无边界深挖

#### `watchlist.md`
观察方向清单。

作用：
- 明确当前重点关注的技术方向
- 划定一级、二级关注范围
- 统一后续巡逻的雷达边界

#### `reddit-radar.md`
Reddit 侧的角色定义文件。

作用：
- 明确 Reddit 负责“热点发现、情绪观察、痛点提取、方向预警”
- 规定优先关注哪些 subreddit
- 说明每次巡逻需要记录什么

#### `github-learning-radar.md`
GitHub 侧的角色定义文件。

作用：
- 明确 GitHub 负责“验证实现、学习结构、吸收方法”
- 规定筛 repo 的标准
- 规定需要重点观察 README、目录结构、issue/discussions、维护状态等

## 当前工作流

### 总工作流
1. 先看 Reddit 最近热帖和评论关键词
2. 找出值得深挖的话题和痛点
3. 再去 GitHub 找对应方向下的真实项目
4. 判断哪些项目有真实结构、文档和使用痕迹
5. 最后提炼可迁移的方法，回灌到当前项目体系

### 当前执行方式
目前主要是 **文档定义 + 最小抓取脚本** 的组合：
- 文档负责定边界和标准
- 脚本负责拉一轮真实素材
- 人负责筛选、判断和总结

所以当前阶段仍是：

**半自动巡逻原型，而不是全自动研究系统。**

## 当前还不具备的能力

以下能力目前 **还没有真正做完**：

- 完整配置文件体系
- 正式环境说明
- 参数化抓取范围
- 自动打分 / 自动排序
- 自动深读某个 repo
- 自动抽取评论高频痛点
- 自动生成结构化周报
- 定时调度与日志体系
- 多轮归档与历史对比
- 一键部署到别的电脑的成熟安装流程

所以对外口径必须克制：

### 可以说
- 这是一个 **可执行的双引擎巡逻原型**
- 已经具备最小真实抓取能力
- 可以帮助建立“发现热点 -> 找实现 -> 回灌方法”的工作流

### 不能说
- 不能说它已经是成熟产品
- 不能说它已经实现全自动研究闭环
- 不能说它已经具备完整报告系统和工程化调度能力

## 最小运行方式

### 环境
- Windows
- Python 3.x
- 能正常访问 Reddit / GitHub 页面

### 执行命令
在项目根目录下运行：

```powershell
python .\self-evolution\scripts\run_real_patrol.py
```

### 运行成功标志
脚本正常执行后，会输出一个本地文件路径，通常为：

```text
C:\Users\besam\.openclaw\workspace\self-evolution\patrol-run-2026-04-17.txt
```

### 输出内容包含什么
- 部分 Reddit 热帖标题与链接
- 部分 GitHub topic 下匹配到的 repo 链接

这份输出的作用是：

**作为人工筛选与后续深读的原始素材。**

## 后续建议升级方向

如果要继续往前做，优先级建议如下：

### 第一优先级：工程化基础
1. 增加 `README.md`（本文件）
2. 增加配置文件，例如：
   - 抓取哪些 subreddit
   - 抓取哪些 GitHub topics
   - 每类抓多少条
3. 增加输出目录规范
4. 增加日志与错误处理

### 第二优先级：结构化分析
1. 自动提取热门关键词
2. 自动标记候选 repo
3. 自动形成“值得深读 / 不值得深读”的初筛结果
4. 自动生成每日巡逻摘要

### 第三优先级：调度与迁移
1. 加计划任务 / scheduler
2. 支持历史归档与对比
3. 写可复制到新电脑的迁移部署教程
4. 增加周报模板和输出脚本
