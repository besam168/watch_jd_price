# HEARTBEAT.md

# 自我进化轻自动执行清单
# 目标：让 self-evolution 用 heartbeat 轻驱动，不靠死板重调度。

- 先看 `self-evolution/README.md` 的总原则与执行节奏。
- 再看 `self-evolution/learning-log.md`、`self-evolution/watchlist.md`、`memory/2026-04-17.md`，避免重复劳动。
- 每次 heartbeat 只做一件最有价值的小事，优先级如下：
  1. 如果最近真实工作暴露了能力缺口（搜索不稳、抓取不稳、skill 卡点、自动化链路不稳），优先补洞，并把修补结果沉淀成脚本、模板、skill 或文档。
  2. 如果没有明显卡点，则做一次轻巡逻：快速看 Reddit / GitHub 是否出现值得记录的新信号。
  3. 如果已经积累了足够信号，则推进一次深读或补一条中文总结。
- 不要为了 heartbeat 硬产出废话；没有实质进展时回复 `HEARTBEAT_OK`。
- 如果做了有效推进：
  - 更新 `self-evolution` 相关文件
  - 追加记录到当天 `memory/YYYY-MM-DD.md`
  - 必要时 git 提交，提交信息保持简洁准确
