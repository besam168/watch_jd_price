# GitHub 学习雷达执行清单

## 双引擎总原则
- **Reddit 负责发现热点与信号**
- **GitHub 负责验证实现与吸收方法**
- 工作顺序默认是：**先看 Reddit 有没有持续热议，再去 GitHub 找真项目深挖**

---

## 日常轻巡逻（每天 1 次）
### 目标
- 看趋势
- 收候选
- 不陷入无效深挖

### Reddit 轻巡逻动作
1. 看重点板块最近热帖标题
2. 看高赞评论里反复出现的关键词
3. 记录用户真实痛点和情绪
4. 判断哪些话题值得去 GitHub 深挖

### GitHub 轻巡逻动作
1. 看近期值得关注的 repo 标题与主题
2. 看 issue / discussions 里是否有真实使用痕迹
3. 记录新出现的关键词
4. 把值得深读的项目写入 `learning-log.md`

### 轻巡逻输出模板
- 日期：
- Reddit 热点：
- Reddit 关键词：
- GitHub 候选 repo：
- 为什么值得看：
- 是否安排深读：是 / 否

---

## 深读（每 2~3 天 1 次）
### 目标
真正吸收 1~2 个项目。

### 固定拆解项
1. 项目解决什么问题
2. 用户是谁
3. 核心结构是什么
4. 文档哪里写得好
5. 最值得抄的 3 个点
6. 有哪些失败场景 / 边界
7. 哪些是包装，哪些是真能力
8. 它是否回应了 Reddit 上出现的真实需求
9. 怎么回灌到我自己或大老板项目

### 深读输出模板
- 日期：
- Repo：
- 方向：
- 对应 Reddit 热点：
- 核心价值：
- 结构亮点：
- 文档亮点：
- 边界 / 风险：
- 可迁移方法：
- 可立即回灌动作：

---

## 周报（每周 1 次）
### 目标
给大老板看结果，不让学习变成自嗨。

### 周报模板
1. 本周 Reddit 上开始火的方向
2. 本周值得看的 GitHub 项目 / 方向
3. 本周学到的结构与方法
4. 我准备升级的能力点
5. 对当前项目可直接借鉴的东西
6. 不值得继续追的坑

## 进入候选池的标准
满足以下至少 3 条：
- 有清晰 README
- 最近仍在维护
- 有真实代码
- 有明确场景
- 文档可读
- 结构值得抄
- 有人真在 issue / discussion 中使用

---

## 立即丢弃的信号
- 只有宣传没有结构
- 没文档
- 没更新
- 全靠 buzzword 堆砌
- 没有可迁移价值
