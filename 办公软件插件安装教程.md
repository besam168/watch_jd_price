# 办公室软件插件安装教程（OpenClaw 另一台电脑适用）

> 适用目标：在另一台 Windows 电脑上的 OpenClaw 中，安装和使用“办公文档类”插件。
> 
> 这份教程按**最实用**来写，不讲虚的，直接说这台环境里目前对应的插件叫什么、用来干什么、怎么放、怎么验证。

---

## 一、先说结论：办公室软件相关插件分别叫什么

当前这套工作区里，和“办公室软件”最相关的插件/skill 主要是这几个：

### 1）Word 文档
#### A. `docx-generator`
- 用途：快速生成简单 Word 文档（`.docx`）
- 适合：文章、报告、说明书、作文、简单格式文档
- 优点：轻、直接、够用，不依赖 Word GUI

#### B. `minimax-docx`
- 用途：更强的 `.docx` 读写、编辑、模板处理、占位符替换、结构分析
- 适合：复杂 Word 文档处理、模板化文档、保格式编辑
- 当前状态：你这台工作区里有完整参考副本，适合复制到另一台电脑单独安装

### 2）Excel 表格
#### `minimax-xlsx`
- 用途：创建、读取、编辑、分析 Excel / `.xlsx` / `.csv`
- 适合：表格、财务模型、名单、统计表、公式校验
- 这是当前“Excel 办公”最核心的插件

### 3）PPT 演示文稿
#### `pptx-generator`
- 用途：生成、编辑、读取 PowerPoint（`.pptx`）
- 适合：汇报 PPT、项目展示、课件、演示稿
- 支持从零创建，也支持结构化生成

### 4）Word 转 PDF
#### `word-to-pdf`
- 用途：把 `.docx` 转为 PDF
- 适合：需要交正式版 PDF 的场景

---

## 二、另一台电脑最建议安装哪些

如果你想在另一台 OpenClaw 上先把“办公室软件能力”装齐，优先顺序建议这样：

### 基础必装
1. `docx-generator`
2. `minimax-xlsx`
3. `pptx-generator`
4. `word-to-pdf`

### 进阶再装
5. `minimax-docx`

原因很简单：
- `docx-generator` 负责**快速生成 Word**
- `minimax-xlsx` 负责**Excel**
- `pptx-generator` 负责**PPT**
- `word-to-pdf` 负责**输出 PDF**
- `minimax-docx` 负责**复杂 Word 编辑与模板化**

这五个装上，另一台机器的办公文档能力就基本成型了。

## 三、这些插件在这台机器里分别在哪里

### 1）本地工作区已有的插件
这些通常可以直接从本工作区复制到另一台电脑：

- `skills\docx-generator\`
- `skills\pptx-generator\`
- `skills\word-to-pdf\`

### 2）Excel / 强 Word 参考来源
当前工作区里可见的参考副本主要在：

- `._tmp\minimax-skills\skills\minimax-xlsx\`
- `._tmp\minimax-skills\skills\minimax-docx\`

如果另一台电脑要装，最稳妥的方法是：
- 先把这两个 skill 目录完整复制出来
- 再放到另一台机器的 OpenClaw `skills/` 目录下

## 四、另一台电脑怎么安装（最稳方法）

下面按 **Windows + OpenClaw** 的最实用方式来。

### 第一步：找到另一台电脑的 OpenClaw 工作区
通常你要找到类似目录：

```powershell
C:\Users\<你的用户名>\.openclaw\workspace
```

进入后，确认里面有：

```powershell
skills\
```

如果没有，就自己创建一个：

```powershell
mkdir C:\Users\<你的用户名>\.openclaw\workspace\skills
```

---

### 第二步：把插件目录复制过去
把下面这些目录复制到另一台电脑的：

```powershell
C:\Users\<你的用户名>\.openclaw\workspace\skills\
```

建议先复制这几个：

```text
skills\docx-generator
skills\pptx-generator
skills\word-to-pdf
._tmp\minimax-skills\skills\minimax-xlsx   → 复制后改放到 skills\minimax-xlsx
._tmp\minimax-skills\skills\minimax-docx   → 复制后改放到 skills\minimax-docx
```

复制完成后，另一台机器最好长这样：

```text
workspace\skills\docx-generator\
workspace\skills\pptx-generator\
workspace\skills\word-to-pdf\
workspace\skills\minimax-xlsx\
workspace\skills\minimax-docx\
```

---

### 第三步：重启 OpenClaw
复制完 skill 后，重启另一台电脑上的 OpenClaw。

如果是命令行环境，通常可以先看状态：

```powershell
openclaw status
```

如果有需要，再重启 gateway：

```powershell
openclaw gateway restart
```

如果那台机器的运行方式和这里不同，就按那台机器自己的 OpenClaw 启动方式重开一次。

## 五、每个插件怎么快速验证

### A. 验证 `docx-generator`
先准备一个 JSON 文件，例如：

```json
{
  "title": "测试文档",
  "author": "OpenClaw",
  "sections": [
    {
      "heading": "第一部分",
      "paragraphs": ["这是第一段。", "这是第二段。"],
      "bullets": ["要点1", "要点2"]
    }
  ]
}
```

然后执行：

```powershell
python skills\docx-generator\scripts\make-docx.py --input test.json --output demo.docx
```

如果生成出 `demo.docx`，并且 Word / WPS 能打开，就说明 OK。

---

### B. 验证 `pptx-generator`
最简单的方式是先确认目录和说明文件都在，然后按 skill 自带说明生成一个最小 PPT。

先看：

```powershell
skills\pptx-generator\SKILL.md
```

如果需要结构化生成 PPT，一般会在该 skill 目录下按它的说明执行 Node / JS 生成逻辑。

重点验证标准：
- 能产出 `.pptx`
- PowerPoint / WPS 演示 能正常打开

---

### C. 验证 `minimax-xlsx`
目标是确认：
- skill 目录完整
- 脚本可运行
- 能生成或读取 `.xlsx`

先看 skill 说明：

```powershell
skills\minimax-xlsx\SKILL.md
```

然后按它的脚本说明跑一个最小测试表。

重点验证标准：
- 能生成 `.xlsx`
- Excel / WPS 表格能正常打开
- 若带公式，打开后公式正常

---

### D. 验证 `minimax-docx`
这是强 Word 插件，验证比 `docx-generator` 稍复杂。

先看：

```powershell
skills\minimax-docx\SKILL.md
```

如果是 Windows，优先看它目录下是否有：

```powershell
scripts\setup.ps1
```

如果有，就先执行初始化：

```powershell
powershell -ExecutionPolicy Bypass -File skills\minimax-docx\scripts\setup.ps1
```

然后再按 SKILL.md 的最小 create / edit / validate 示例跑通。

---

### E. 验证 `word-to-pdf`
准备一个 `.docx`，然后按 skill 的说明转 PDF。

重点看：
- 能生成 PDF
- 中文不乱码
- 版式基本正常

## 六、另一台电脑可能缺的环境

安装办公插件时，另一台机器最可能缺的是这些：

### 1）Python
很多本地 skill 都依赖 Python。
先验证：

```powershell
python --version
```

如果没有，就先装 Python。

---

### 2）Node.js
`pptx-generator` 这类常常会用到 Node。
先验证：

```powershell
node --version
npm --version
```

如果没有，就装 Node.js LTS。

---

### 3）LibreOffice（可选但很有用）
如果涉及 `.doc` 转 `.docx`、部分格式转换、Office 兼容补充，建议装。

Windows 可用：

```powershell
winget install TheDocumentFoundation.LibreOffice
```

---

### 4）Microsoft Office 或 WPS
虽然这些插件很多可以**不依赖 Office GUI 生成文件**，但你最终还是需要：
- Microsoft Word / Excel / PowerPoint
或
- WPS Office

来人工打开验收。
