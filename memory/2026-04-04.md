## Whisper skill / STT 层决策补记

- 大老板后续指定把 `Openai Whisper v1.0.0`（`Local speech-to-text with the Whisper CLI (no API key)`）作为参考并要求尽快纳入项目。
- 当前技术判断已经明确：
  - 它的本质不是 TTS，不是整套语音助手，而是**本地 STT（语音转文字）能力封装**；
  - 对 `tmall-genie-voice-bridge` 的参考价值**高于很多花哨 TTS 方案**，因为它正对应当前项目最核心的一层：`mic -> audio -> text`；
  - 最合适的角色是：**STT provider 候选 / 标准化本地 Whisper 接口**，而不是把它硬写进业务逻辑里。
- 已形成新的实施原则：
  - `tmall-genie-voice-bridge` 未来应按 provider 分层：`wake provider / stt provider / tts provider / speaker provider`；
  - `Openai Whisper` 应归入 **STT provider**；
  - `edge-tts / aliyun-tts / minimax-free-api` 这类方案应归入 **TTS provider**；
  - 不能再把“听、说、路由、动作”写死在单脚本里。
- 大老板已进一步要求：**现在就把它加入，并做好马上测试**。
- 因此从这一刻起，这条线的目标不再是纯架构讨论，而是：
  1. 把 Whisper skill / 本地 Whisper 能力正式纳入项目的 provider 设计；
