## nano-banana-bridge 生图插件升级记录

- 今天已把 `skills/nano-banana-bridge/` 从“仅 mock 骨架”推进到 **可真实调用 OpenAI-compatible 图像接口** 的可用版本。
- 当前已真实接通的接口参数：
  - Base URL：`https://api-cn.hi-code.cc/v1`
  - Model：`gpt-image-1`
  - 调用方式：`POST /images/generations`
  - 响应已验证支持：`data[0].b64_json`
- 已真实出图成功，不再是空壳 mock。

## 当前已支持的能力

### 1. 调用模式
- `--provider openai-compatible`
- `--base-url`
- `--api-key`
- `--model`
- 也支持从环境变量读取：
  - `OPENAI_IMAGE_BASE_URL`
  - `OPENAI_IMAGE_API_KEY`
  - `OPENAI_IMAGE_MODEL`

### 2. 尺寸输入方式
支持两种：
1. **显式尺寸**
   - 例如：`--size 2048x1152`
2. **比例 + 分辨率档位**
   - 例如：`--aspect-ratio 16:9 --resolution 2K`

### 3. 已支持的比例（8种）
- `1:1`
- `3:2`
- `2:3`
- `16:9`
- `9:16`
- `4:3`
- `3:4`
- `21:9`

### 4. 已支持的分辨率档位（3种）
- `1K`
- `2K`
- `4K`

## 当前内置映射
- `1:1` → `1024x1024` / `2048x2048` / `4096x4096`
- `3:2` → `1536x1024` / `2304x1536` / `4608x3072`
- `2:3` → `1024x1536` / `1536x2304` / `3072x4608`
- `16:9` → `1792x1024` / `2048x1152` / `4096x2304`
- `9:16` → `1024x1792` / `1152x2048` / `2304x4096`
- `4:3` → `1365x1024` / `2731x2048` / `5461x4096`
- `3:4` → `1024x1365` / `2048x2731` / `4096x5461`
- `21:9` → `2389x1024` / `4779x2048` / `9557x4096`

## 当前边界
- 目前优先打通的是 **文生图**。
- `input-image` 图生图/编辑分支在 `openai-compatible` 模式下还未实现。
- `/v1/responses + image_generation` 兼容层也还没补。
- 4K 映射值已写入 bridge，但不同尺寸是否都被上游接口接受，仍要以真实返回为准；不能先吹全支持。

## 使用建议
- 要稳定出图时，优先用：
  - `--aspect-ratio 16:9 --resolution 2K`
  - 或直接 `--size 2048x1152`
- 做学校项目图、比赛展板、流程示意图时，这条链路已经能正式用。
