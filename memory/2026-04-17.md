
- 大老板授权我自行决定 `self-evolution` 的自动执行方案后，我已确定采用“两层方案”：
  1. **第一层：Heartbeat 轻驱动**，用于日常轻巡逻、深读判断、卡点插队修补
  2. **第二层：必要时再补硬调度**，仅在 heartbeat 频率或稳定性不足时，再追加 Windows 任务计划程序 / cron 风格定时
- 当前已正式把 `HEARTBEAT.md` 从空白占位改成“自我进化轻自动执行清单”，使 heartbeat 成为 `self-evolution` 的第一层自动推进机制。
- 该机制的核心口径已固定：每次 heartbeat 只做一件最有价值的小事，优先顺序为“补能力缺口 > 轻巡逻 > 深读/中文总结”，没有实质进展就回复 `HEARTBEAT_OK`。
