  2. 支持 `/images/generations` 与 `/responses + image_generation` 两条路径

  4. 新增 OpenAI 常见合法尺寸规整逻辑（例如把高风险尺寸规整到 `1024x1024 / 1536x1024 / 1024x1536`）

- 对不同路由做了大量真实踩坑验证：
  - `https://www.hi-code.cc/v1`
    - `gpt-5.3-codex`、`gpt-image-1`、`gpt-image-2` 均返回 `HTTP 403 / error code 1010`
    - 判定为当前图像请求被风控/不可用
  - `https://api-cn.hi-code.cc/v1` + `/responses`
    - 不会 403，但会退化成普通文本输出
    - 返回中可见 `tools: []` / `tool_choice: none`
    - 说明这条兼容层不能把 Responses 当正式生图路径
  - `https://api-cn.hi-code.cc/v1` + `/images/generations`
    - 多次测试后确认这是当前真实可用主链路
- 成功出图实绩：

     - `skills/nano-banana-bridge/output/nano-banana_20260429_162355.png`

- 首次用普通 GitHub SSH 22 端口推送失败：
  - 报错表现为连接被关闭
- 改走 GitHub SSH over 443 路线：
  1. 先执行：`ssh -T -p 443 git@ssh.github.com`
  2. 人工确认 `yes`，将 host key 写入 `known_hosts`
  3. 再将 remote 设为：
     - `ssh://git@ssh.github.com:443/besam168/office-productivity-skills.git`
  4. 成功 `git push -u origin main`
- 形成新的长期 Git 经验：
  - 这台机器上传 GitHub **优先 SSH，不走 HTTPS**
  - 若 22 端口 SSH 不通，**立刻切 `ssh.github.com:443`，不要死磕**
- 该经验已写入：
  - `MEMORY.md`
  - `TOOLS.md`
