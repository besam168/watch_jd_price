- 已确认旧问题根因：`local_windows_speaker` 旧链路里，`WMPlayer` 对本地 WAV 不可靠，`playState=9` 不能当真人播放成功。
- 已在 `skills/tmall-genie-voice-bridge/scripts/play-local-audio.ps1` 中为 `.wav` 增加 `System.Media.SoundPlayer.PlaySync()` 快路径，修掉“假成功 / 假失败”问题。
- 已实测通过：
  - `speak-local.ps1 -Text "你好"`
  - `scripts/speak.py --config config.local-speaker.json`
  - `bridge_server.py --config config.local-speaker.json`
  - `POST /speak` 触发本地扬声器播报
- `speak-local.ps1` 已补 UTF-8 / 中文参数处理，中文文本与 JSON 输出链路比之前稳定。
- 新增最小文本演示脚本：
  - `skills/tmall-genie-voice-bridge/demo-text-roundtrip.ps1`
  - 支持 `-Mode local` 与 `-Mode bridge`
